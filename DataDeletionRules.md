# Data Deletion Rules for PorosenoCheck Employee App

## Overview
This document outlines the rules and procedures for data deletion requests in the PorosenoCheck Employee App, as required by Google Play Store policies.

## Rule 1: Data Deletion Request Process

### Error Resolution
- Fix DNS error "Имя не найдено в DNS" by ensuring the data deletion request page URL is valid and accessible
- Implement proper domain name resolution for the data deletion request page

### Implementation Requirements
1. Users must be able to access the data deletion request page directly from the app
2. The URL must be a valid, publicly accessible web page
3. The page must allow users to submit data deletion requests
4. DNS settings must be properly configured to resolve the domain name

### User Interface Requirements
1. Include a "Delete Data" option in the app settings
2. Provide clear instructions for the data deletion request process
3. Ensure the link opens the correct deletion request page in a web browser

## Rule 2: Data Deletion Request Functionality

### Request Process
1. Users can request deletion of specific data through the app interface
2. Data deletion requests are processed within 30 days
3. Users can request deletion of:
   - Personal profile information
   - Usage history
   - Location data
   - Communication records

### Data Handling
1. Upon deletion request, specified personal data is removed from active databases
2. Backup data is deleted according to retention policies
3. Anonymous usage data may be retained for analytical purposes
4. Data that is required for legal or business purposes may be retained

### Compliance
1. Comply with GDPR, CCPA, and other applicable data protection laws
2. Provide clear information about the data deletion process in the privacy policy
3. Allow users to understand what data will be retained and why
4. Ensure the deletion process is properly documented and auditable
